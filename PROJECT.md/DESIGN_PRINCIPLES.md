# Design Principles

Este documento define cómo diseñamos Språkkafé Oslo.

Siempre que exista una duda sobre una nueva funcionalidad, diseño o contenido, este documento tiene prioridad.

---

# Nuestra pregunta

Antes de implementar cualquier cambio debemos preguntarnos:

> ¿Esto ayuda a que una persona se sienta parte de Noruega?

Si la respuesta es sí...

Seguimos adelante.

Si la respuesta es no...

No pertenece al proyecto.

---

# 1. La comunidad primero

El objetivo del proyecto no es mostrar información.

El objetivo es conectar personas.

Todo debe facilitar que alguien encuentre un lugar donde conversar.

---

# 2. Simplicidad

Menos siempre es más.

Preferimos eliminar elementos antes que añadir nuevos.

Cada pantalla debe tener un único objetivo.

---

# 3. Claridad

La información debe entenderse en pocos segundos.

Evitamos:

- bloques largos de texto
- duplicidad de información
- navegación innecesaria

---

# 4. Confianza

La información debe transmitir seguridad.

Por ello:

- mostramos organizaciones verificadas
- enlazamos siempre a la fuente oficial
- evitamos información dudosa

---

# 5. Accesibilidad

Muchos usuarios todavía están aprendiendo noruego.

Por eso:

- lenguaje sencillo
- botones grandes
- buen contraste
- navegación intuitiva
- diseño responsive

---

# 6. Espacio en blanco

El espacio también comunica.

Preferimos separar mediante padding antes que utilizar líneas divisorias.

---

# 7. Jerarquía visual

Cada pantalla debe responder inmediatamente:

¿Qué es esto?

¿Dónde estoy?

¿Qué puedo hacer ahora?

Nunca deben competir varios elementos importantes entre sí.

---

# 8. Microinteracciones

Todo elemento interactivo debe responder.

Hover.

Focus.

Animaciones suaves.

Nada debe sorprender al usuario.

---

# 9. Mobile First

La mayoría descubrirá actividades desde el móvil.

Diseñamos primero para móvil.

Después mejoramos la experiencia en escritorio.

---

# 10. El contenido manda

El protagonista nunca es la interfaz.

El protagonista son:

- las actividades
- las organizaciones
- las personas

El diseño debe desaparecer.

---

# 11. Escalabilidad

Hoy construimos para Oslo.

Cada decisión debe permitir crecer hacia:

- Bergen
- Trondheim
- Stavanger
- Tromsø
- toda Noruega

Sin tener que rediseñar el producto.

---

# 12. Nuestra filosofía

No somos una academia.

No organizamos eventos.

No sustituimos a las organizaciones.

Somos el puente entre las personas y la comunidad.

---

# Regla de oro

Si una nueva funcionalidad hace el producto más complejo...

...debe aportar mucho más valor del que añade en complejidad.

En caso contrario, no se implementa.
